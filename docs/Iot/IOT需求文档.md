# IoT 物联网平台产品功能需求文档

## 项目概述

物联网平台是一个综合性的云端系统，参考阿里云物联网平台架构，用于管理和监控各种物联网设备。平台提供设备接入、设备管理、数据通信、数据存储、规则引擎、设备影子、OTA升级、安全认证等核心功能。该平台将支持MQTT、CoAP、HTTP等多种通信协议，为用户提供统一的设备管理界面、数据可视化能力和业务集成接口。

## 功能优先级分级

### 🔴 P0 - 核心基础功能（必须实现）
**目标**：构建平台基础能力，实现基本的设备管理和数据通信
- 设备与产品管理（基础功能）
- 设备通信与数据管理（基础协议）
- 用户权限管理（基础RBAC）
- 基础监控告警
- 系统管理与配置（基础配置）

### 🟡 P1 - 核心业务功能（重要实现）
**目标**：完善平台核心业务能力，提供完整的IoT解决方案
- 规则引擎与数据转发
- 数据分析与可视化（基础图表）
- 多协议接入扩展
- 高级权限与多租户
- 运维监控完善

### 🟢 P2 - 增值功能（优化实现）
**目标**：提升平台竞争力和用户体验
- 平台开放与生态集成
- 高级数据分析
- 边缘计算基础
- 移动端与小程序
- 行业解决方案

### 🔵 P3 - 创新功能（探索实现）
**目标**：技术前瞻性和差异化竞争
- AI能力集成
- 数字孪生
- 区块链集成
- 大模型应用
- 新兴技术探索

## 一、设备与产品管理 🔴 P0

### 基础功能（P0）
- **产品分类管理**
    - 产品分类的创建、编辑、删除、移动
    - 分类树结构管理，支持多级分类（如：智能家居/照明设备/智能灯泡）
    - 分类模板管理，预置标准行业分类模板
    - 分类权限控制，支持分类级别的数据隔离
- **产品管理**
    - 产品的创建、编辑、删除、复制、批量操作
    - 产品基本信息：唯一标识（identifier）、名称、节点类型（直连/网关/子设备）、认证方式（密钥/证书/免认证）
    - 产品分类归属，支持产品在分类树中的归属管理
    - 产品状态管理（开发中/已发布/已下线）
    - 产品使用统计（设备数量、消息量、在线时长等）
- **产品模板管理**
    - 产品模板的创建、编辑、删除、复制
    - 模板包含预定义的产品配置和物模型
    - 基于模板快速创建产品，支持模板继承和自定义修改
    - 标准行业模板库（智能家居、工业设备、农业设备等）
    - 模板版本管理和模板发布流程
- **物模型（Thing Model）管理**
    - 物模型模板的创建、导入、导出、启用、禁用
    - 属性、事件、服务的标准TSL定义
    - 物模型与产品的绑定、版本管理
    - 物模型模板库，支持从模板库中选择和继承物模型
- **设备管理**
    - 设备注册、批量注册、注销、禁用、激活
    - 设备信息（名称、标签、分组、状态、元数据）管理
    - 设备状态实时监控（在线/离线/故障/维护）
    - 设备唯一身份认证（如证书、Token）
- **设备分组与标签管理**
    - 分组树结构、分组的增删查改
    - 设备与分组、标签的多对多绑定/解绑
    - 分组权限隔离与可见性控制

### 高级功能（P1-P2）
- **产品分类高级管理**
    - 分类数据统计与分析（产品数量、设备数量、活跃度等）
    - 分类权限继承与委托机制
    - 分类标签与元数据管理
    - 分类模板市场，支持第三方分类模板导入
    - 分类数据迁移与合并功能
- **产品高级管理**
    - 支持产品多协议（如LoRa、NB-IoT、Modbus、CoAP、LwM2M等）能力扩展
    - 产品批量导入/导出（Excel/CSV/JSON）
    - 产品生命周期管理（开发、测试、发布、下线、归档）
    - 产品配置继承与差异化管理
    - 产品使用统计与分析报告
    - 产品性能监控与优化建议
- **产品模板高级管理**
    - 模板市场与模板分享机制
    - 模板评分与评论系统
    - 模板使用统计与热门模板推荐
    - 模板版本对比与差异分析
    - 模板权限控制与访问管理
    - 自定义模板创建向导
- **物模型高级功能**
    - 物模型属性支持多类型（数值、枚举、结构体、数组、地理位置等）
    - 属性单位、精度、校验规则自定义
    - 物模型变更历史追踪与回滚
    - 物模型可视化编辑器（拖拽式）
    - 物模型校验与测试工具
    - 标准行业物模型库（智能家居、工业4.0、智慧城市等）
    - 物模型继承与扩展机制
- **设备高级管理**
    - 标签支持多维度（如地理、用途、厂商、环境等）扩展
    - 动态分组（基于规则自动分组）
    - 分组模板与快速应用
    - 分组统计与分析
    - 设备远程诊断与日志采集
    - 设备批量导入/导出、批量操作任务队列
    - 设备地理位置管理与地图展示（支持室内定位）
    - 设备远程调试与诊断（如远程日志、远程Shell、摄像头画面、性能监控）
    - 设备固件差分升级与自愈机制
    - 设备行为建模与自动异常告警
    - 设备资产管理（采购、维保、折旧）
    - 设备二维码/条形码管理
    - 设备配置模板与批量配置
    - 设备影子（Device Shadow）状态管理
    - 设备拓扑关系管理（父子设备、网关设备）

## 二、设备通信与数据管理 🔴 P0

### 基础功能（P0）
- **基础协议接入**
    - 支持 MQTT、HTTP/HTTPS、WebSocket 等主流协议
    - 设备认证与安全接入（X.509证书、PSK）
    - 设备接入限流与黑白名单
- **消息上下行与 Topic 管理**
    - 设备属性、事件上报，平台命令下发
    - Topic 模板定义、类型与操作类型管理
    - 消息格式（JSON、二进制）与校验规则
    - 支持 QoS 等级、消息重试与补发
- **数据持久化与查询**
    - 设备数据持久化存储（支持结构化数据）
    - 历史数据查询与导出（支持多种格式）
    - 支持MySQL、Redis基础存储

### 高级功能（P1-P2）
- **多协议接入扩展**
    - CoAP、LwM2M、TCP/UDP 等协议支持
    - 工业协议支持（Modbus RTU/TCP、OPC UA、BACnet、DNP3、IEC 61850等）
    - 无线协议支持（LoRa、NB-IoT、2G/3G/4G/5G、WiFi、蓝牙、Zigbee等）
    - OAuth2.0认证支持
    - 支持自定义协议适配器插件（热插拔）
    - 跨协议网关与自动协议转换
    - 协议适配器管理与监控
    - 设备接入负载均衡与故障转移
- **高级消息处理**
    - Topic 权限与订阅管理
    - 消息追踪与链路监控
    - 消息路由与分发策略
    - 消息压缩与批量处理
    - 消息去重与幂等性保证
    - 实时消息推送（WebSocket、SSE）
    - 消息格式扩展（Protobuf、Avro）
- **高级数据管理**
    - 数据分库分表、冷热数据分层存储
    - 数据清理与归档策略（自动化生命周期管理）
    - 支持多种数据库（PostgreSQL、MongoDB、InfluxDB、TDengine、ClickHouse等）
    - 时序数据压缩与高效检索
    - 地理围栏与轨迹分析
    - 数据血缘与影响分析
    - 数据质量监控与异常检测
    - 流式数据处理（Apache Kafka、Apache Pulsar）
    - 数据湖与数据仓库集成
- **数据处理与计算**
    - 实时流计算（Apache Flink、Apache Storm）
    - 批量数据处理（Apache Spark）
    - 数据清洗与标准化
    - 数据聚合与统计计算
    - 复杂事件处理（CEP）
    - 数据挖掘与机器学习集成

## 三、用户、权限与多租户 🔴 P0

### 基础功能（P0）
- **用户管理**
    - 用户注册、登录、登出、信息维护
    - 支持第三方登录（OAuth2、LDAP、AD）
    - 用户操作审计与安全策略（如密码强度、登录限制、会话管理）
- **权限与角色管理**
    - 基于角色的访问控制（RBAC）
    - 资源细粒度权限分配
    - 细粒度API权限、数据权限、功能菜单权限
- **基础多租户**
    - 企业/组织隔离，子账号与部门管理
    - 租户自定义Logo、域名、配色、主题

### 高级功能（P1-P2）
- **高级用户管理**
    - 支持企业微信、钉钉、飞书等企业登录
    - 多因子认证（MFA）：短信、邮箱、TOTP、硬件令牌
    - 单点登录（SSO）与统一身份认证
    - 用户画像与行为分析
    - 用户生命周期管理（激活、冻结、注销）
    - 密码策略与定期更换提醒
- **高级权限管理**
    - 基于属性的访问控制（ABAC）
    - 角色模板与批量授权
    - 权限继承与委托机制
    - 临时权限与时效性控制
    - 权限审批工作流
    - 权限变更审计与回滚
    - 最小权限原则实施
- **高级多租户功能**
    - 组织结构多级嵌套与灵活调整
    - 多租户计费与用量统计
    - 租户资源配额管理
    - 跨租户数据共享与隔离策略
    - 租户级别的功能开关
    - 租户数据备份与恢复
    - 租户迁移与数据导出
- **企业集成**
    - 企业通讯录同步
    - 工作流集成（审批、通知）
    - 企业应用集成（OA、ERP、CRM）
    - API网关与服务治理

## 八、安全、合规与智能扩展 🟡 P1

## 十五、设计文档编写要求 📋

### 15.1 设计文档概述

设计文档是IoT平台开发的重要指导文档，用于详细描述系统架构、技术方案和实现细节。

### 15.2 注意事项

#### 15.2.1 技术选型注意事项
- 选择成熟稳定的技术栈
- 考虑团队技术栈和学习成本
- 支持未来扩展需求
- 说明技术选型理由

#### 15.2.2 架构设计注意事项
- 使用 mermaid 画图工具绘制架构图
- 基于Spring Modulith进行模块化设计
- 模块间保持松耦合，便于后续微服务拆分
- 采用技术分层架构（controller/service/mapper/entity/dto/config）

#### 15.2.3 数据库设计注意事项
- 详细说明数据库选型理由
- 重点设计核心业务表结构

#### 15.2.4 接口设计注意事项
- 遵循RESTful API设计规范
- 完整的参数校验和错误处理
- Controller和Service接口设计，不考虑实现细节
